## Situation

the probe can be used to edit field of any instance.

fields are "gong fields", meaning simple types, pointers and slice of pointers.

but probe allows also to edit "reverse" alice of pointers.

in test/test3/go/probe

if you edit a B instance, there is a within the form, a form field select field pull to edit what A instance field "Bs" contains this B instance.

test/test3/go/probe/fill_up_form.go

```go
		{
			var rf models.ReverseField
			_ = rf
			rf.GongstructName = "A"
			rf.Fieldname = "Bs"
			reverseFieldOwner := instanceWithInferedType.GongGetReverseFieldOwner(probe.stageOfInterest, &rf)
			if reverseFieldOwner != nil {
				AssociationReverseFieldToForm(
					reverseFieldOwner.(*models.A),
					"Bs",
					instanceWithInferedType,
					formGroup,
					probe)
			} else {
				AssociationReverseFieldToForm[*models.A](
					nil,
					"Bs",
					instanceWithInferedType,
					formGroup,
					probe)
			}
		}
```

test/test3/go/probe/association_to_form.go

## problem

this form makes works well if in the model, the Bs field is a composition (liek 0..1-N cardinality) but gong
slice of pointer fields have only 0..N-O..N cardinality.

One should be able to select wich A instances have this B instance in their "Bs" slice of pointers field.

## solution

instead of using a form select, the probe should use something similar to AssociationSliceToForm to select wich A instances have this B instance in their "Bs" slice of pointers field

the setup of AssociationSliceToForm have been a complex endeavor, so this solution is not easy to implement.
